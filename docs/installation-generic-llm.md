# Подключение к любому ИИ-чату (Gemini, Llama, DeepSeek, Mistral, ...)

Если ты используешь LLM, у которого нет родного формата "скиллов" (`SKILL.md` с frontmatter) и нет project knowledge — есть простой путь: **собери все нужные карточки в один markdown-файл и вставь его в начало диалога**.

Это работает с любой моделью, которая принимает достаточно длинный контекст (8k+ токенов).

## Шаг 1. Соберём bundle

В репо есть скрипт `scripts/bundle.py`, который склеивает `SKILL.md` + выбранные карточки в один файл:

```bash
# собрать всё (SKILL.md + все 5 коннекторов)
python scripts/bundle.py > bundle.md

# только нужный коннектор — экономит токены
python scripts/bundle.py --connector yt-api > yt-bundle.md

# несколько
python scripts/bundle.py -c telegram-channel -c threads-api4 > social-bundle.md

# посмотреть что доступно
python scripts/bundle.py --list
```

Размер bundle.md (для оценки):

| Что | ≈ символов | ≈ токенов |
|---|---|---|
| SKILL.md только | 12 KB | ~3000 |
| SKILL.md + один коннектор | 40-60 KB | ~10-15k |
| Все 5 коннекторов | ~170 KB | ~40k |

**Совет:** не вставляй сразу все 5 коннекторов — это съест контекст. Бери только нужный для текущей задачи.

## Шаг 2. Вставь в чат

### Вариант A: одним сообщением

```
[Скопируй сюда всё содержимое bundle.md]

---

Мой вопрос: напиши Python-функцию, которая ищет видео через yt-api по ключевому слову.
```

ИИ прочитает всю инструкцию + карточку и ответит с правильными хедерами и эндпоинтами.

### Вариант B: через system prompt (если интерфейс позволяет)

Многие LLM-чаты дают возможность задать "system instructions" / "system prompt". Туда можно положить bundle.md один раз, и он будет действовать на весь диалог:

- **Gemini Advanced** — настройки → **Custom Gems** → создаёшь Gem → в "Instructions" вставляешь bundle.
- **Llama / Mistral / DeepSeek** в собственном веб-чате — обычно есть поле "system prompt" в настройках чата.
- **Локальные модели** через Ollama / LM Studio — поле system prompt в UI или ключ `system` в API.

### Вариант C: префиксом в каждом сообщении (костыль)

Если ИИ "забывает" контекст после нескольких сообщений (короткое окно или длинный диалог) — можно добавлять короткое напоминание в начало каждого нового вопроса:

```
Контекст: используй карточку yt-api.md из загруженного bundle.

Вопрос: ...
```

## Что важно учитывать

### 1. Модель должна принимать длинный контекст

| Модель | Контекст | Bundle помещается? |
|---|---|---|
| Gemini 1.5 Pro / 2.0 | 1M+ токенов | ✅ всё легко |
| Claude (любой) | 200k токенов | ✅ всё легко |
| GPT-4o, GPT-4 Turbo | 128k токенов | ✅ всё легко |
| Llama 3.1 70B / 3.3 | 128k | ✅ |
| Llama 3 8B (старая) | 8k | ⚠️ только один коннектор |
| Mistral Large | 128k | ✅ |
| DeepSeek V3 / R1 | 64k+ | ✅ |
| GPT-3.5 | 16k | ⚠️ только один коннектор + укоротить |

Если модель короткоконтекстная — используй `--connector` чтобы взять только нужный.

### 2. ИИ может проигнорировать инструкцию

В отличие от настоящих скиллов, тут нет автоматической активации. Если нейронка не сослалась на карточку и **галлюцинирует** имена параметров — попроси явно:

> Перечитай bundle. Возьми параметры точно из карточки yt-api, не выдумывай.

### 3. Длина bundle vs стоимость

Каждый запрос с большим bundle = больше токенов = больше денег (если LLM платный по токенам). Решения:

- Бери только нужный коннектор (`-c yt-api`).
- Используй модель с **prompt caching** (Anthropic, Gemini, Mistral) — bundle кэшируется, повторные запросы дешевле. См. [installation-via-api.md](installation-via-api.md) для примеров.

## Проверка

Открой свой чат, вставь `bundle.md` (или его часть), затем:

> Какие эндпоинты есть у YT-API для работы с каналами? Перечисли с реальными путями из карточки.

Правильный ответ включит `/channel/about`, `/channel/videos`, `/channel/shorts`, `/channel/playlists`, `/channel/community`, `/channel/channels` и т.д. Если нейронка пишет общие слова без точных путей — она не прочитала карточку.

## Что не работает в этом режиме

- Автоматическая активация скилла по триггерам (только Claude Code и Cursor).
- Авто-подгрузка только нужной карточки — если задачи разноплановые, придётся прикладывать большой bundle.
- Долговременная "память" между сессиями — каждый новый чат = вставлять заново.

Для серьёзной работы с RapidAPI лучше [Claude Code](installation-claude-code.md) или [Cursor](installation-cursor.md). Generic-чат — это запасной вариант "и так сойдёт".
