# Cost modeling — экономика контент-завода

> 🎯 **Цель:** заранее посчитать сколько стоит содержание контент-завода. Чтобы знать когда выгодно перейти на следующий план, сколько авторов реально потянуть, и как масштабироваться без сюрпризов в счетах.

## Главная формула: стоимость одного "миллионника"

Бизнес-вопрос: **сколько мне будет стоить найти 1 виральное видео с миллионом просмотров?**

### Параметры

| Параметр | Значение по умолчанию | Где меняется |
|---|---|---|
| Кол-во авторов в мониторинге | N | твой выбор |
| Постов на автора в день | 1 | реальная статистика по нише |
| Доля виральных от всех (%) | 5% | разная по нише: 1-15% |
| Доля миллионников от виральных | 1% | 0.1-5% в зависимости от размера авторов |
| Запросов на автора в сутки | 25 | 1 профиль + 24 ленты |

### Формула

```
постов_в_месяц = N_авторов × 30 дней × 1 пост/день
виральных_в_месяц = постов_в_месяц × 0.05
миллионников_в_месяц = виральных_в_месяц × 0.01

запросов_в_месяц = N_авторов × 750
api_стоимость = запросов_в_месяц × $/req

стоимость_1_миллионника = api_стоимость / миллионников_в_месяц
```

### Пример: Instagram Looter2, Pro план

```
N_авторов = 20 (бюджет Pro $9.99 = 15k req)
постов/мес = 20 × 30 = 600
виральных/мес = 600 × 0.05 = 30
миллионников/мес = 30 × 0.01 = 0.3 (один раз в ~3 месяца)

стоимость_1_миллионника = $9.99 / 0.3 = $33.30
```

20 авторов на Pro дают 1 миллионник за **~$33** — но раз в 3 месяца. Если нужен каждый месяц — переходи на Ultra (75k req, ~100 авторов).

### Пример: TikTok api23, Ultra план

```
N_авторов = 1333 (Ultra $49.99 = 1M req)
постов/мес = 1333 × 30 × 1.5 (TikTok-блогеры постят чаще) = 60k
виральных/мес = 60k × 0.07 (TikTok вируалит чаще) = 4 200
миллионников/мес = 4200 × 0.02 (TikTok легче пробивает M) = 84

стоимость_1_миллионника = $49.99 / 84 = $0.60
```

TikTok радикально дешевле в пересчёте на миллионник — другая природа платформы.

## Сколько авторов под цель "1 миллионник в месяц"

Обратная формула:

```
N_авторов_нужно = 1 / (30 дней × посты_в_день × P(виральн) × P(миллион))
```

| Платформа | P(виральн) | P(миллион) | Постов/день | N_авторов |
|---|---|---|---|---|
| Instagram | 5% | 1% | 1 | **67 авторов** |
| TikTok | 7% | 2% | 1.5 | **16 авторов** |
| YouTube Shorts | 5% | 0.5% | 0.7 | **190 авторов** |
| Threads | 3% | 0.1% | 2 | **555 авторов** ⚠️ |

> 💡 **Threads** — самая медленная платформа для миллионников (ещё растущая аудитория). Для гарантированного потока — нужен большой портфель авторов.

## Полная стоимость pipeline на одно виральное видео

Считаем все этапы для разных стеков моделей.

### Бюджетный стек (DeepSeek v4 Flash + Replicate Whisper + R2)

| Этап | Стоимость | Примечание |
|---|---|---|
| Мониторинг (амортизирован) | $0.0005 | 25 запросов/автор делятся на много видео |
| Скачивание | $0.005 | 1 запрос download API + bandwidth |
| Транскрипция (Replicate Whisper) | $0.0008 | 1 минута видео × $0.0008/мин |
| Hook-анализ (gpt-5-nano) | $0.00005 | 200 input + 50 output токенов |
| Классификация (gemini-2.0-flash-lite) | $0.0001 | 500 input + 100 output |
| LLM-переписывание (deepseek-v4-flash) | $0.0007 | 3k input + 1k output токенов |
| Хранение в R2 (за 1 видео в месяц) | $0.0001 | ~5 MB × $0.015/GB |
| Уведомление (Telegram) | $0 | бесплатно |
| **Итого на видео** | **~$0.007** | |

**При потоке 100 виральных в день**: 100 × 30 × $0.007 = **~$21/мес**.
**При потоке 1000 в день**: **~$210/мес**.

### Стандартный стек (GPT-4o-mini + OpenAI Whisper + AWS S3)

| Этап | Стоимость |
|---|---|
| Мониторинг | $0.0005 |
| Скачивание | $0.005 |
| Транскрипция (Whisper API) | $0.006 |
| Hook-анализ (gpt-5-nano) | $0.00005 |
| Классификация (gpt-4o-mini) | $0.0006 |
| LLM-переписывание (gpt-4o-mini) | $0.001 |
| Хранение в S3 | $0.0001 |
| **Итого на видео** | **~$0.013** |

**1000 в день** = **~$390/мес**.

### Премиум стек (Claude Sonnet 4.6 с кэшем + Whisper API + R2)

| Этап | Без кэша | С prompt caching (Anthropic) |
|---|---|---|
| Мониторинг | $0.0005 | $0.0005 |
| Скачивание | $0.005 | $0.005 |
| Транскрипция (Whisper) | $0.006 | $0.006 |
| Hook-анализ | $0.00005 | $0.00005 |
| Классификация | $0.001 | $0.001 |
| LLM-переписывание (sonnet 4.6) | $0.025 | **$0.0025** (-90%) |
| Хранение | $0.0001 | $0.0001 |
| **Итого на видео** | **~$0.038** | **~$0.015** |

С кэшем premium стек дешевле стандартного (потому что Claude Sonnet кэшируется, а GPT-4o-mini нет).

**1000 в день**: без кэша $1140/мес, с кэшем $450/мес.

> 💡 **Главная экономия — prompt caching**. Если у тебя 1000+ запросов с тем же системным промптом — Claude Sonnet с кэшем дешевле GPT-4o-mini без.

См. [llm-models.md](llm-models.md) для деталей по моделям и [../installation-via-api.md](../installation-via-api.md) для prompt caching.

## Бюджеты под объём

### Стартап / личный проект

- **20 авторов Instagram + 50 авторов TikTok**.
- Pro-планы обоих коннекторов = $9.99 + $9.99 = $20.
- Ожидаемых виральных: ~30 + ~50 = 80/мес.
- LLM/Whisper на эти 80: ~$3.
- **Итого: ~$23/мес.**

### Малый бизнес (агентство)

- 100 Instagram + 300 TikTok + 50 YouTube.
- Ultra-планы: $27.90 (IG) + $49.99 (TT) + $51 (YT) = $129.
- Ожидаемых виральных: ~150 + ~300 + ~70 = 520/мес.
- LLM/Whisper: ~$20.
- **Итого: ~$149/мес.**

### Крупная редакция

- 300 Instagram + 1000 TikTok + 200 YouTube + 50 Threads.
- Mega-планы: $75.90 (IG) + $99.99 (TT) + $144 (YT) + $19.99 (Threads) = $340.
- Ожидаемых виральных: ~450 + ~1000 + ~280 + ~10 = 1740/мес.
- LLM/Whisper: ~$70.
- Storage S3 для виральных видео (~5MB × 1740 × 6 мес retention): ~$1.20.
- **Итого: ~$420/мес.**

## Где можно сэкономить

### 1. Кэш для повторяющихся запросов

`requests-cache` на 50-55 минут в дев-режиме — экономия до **90%** запросов в день разработки.

В продакшне кэшируй:
- Профили авторов (раз в сутки).
- `/resolve`-конвертеры (handle ↔ id) — навсегда.
- Конвертеры media URL ↔ ID — навсегда.

### 2. `fields` параметр (Instagram looter2)

Сокращает bandwidth на 70-90%. Если у тебя bandwidth-overage > $5/мес — уже окупает время на внедрение.

### 3. Multi-id batch (yt-api, tiktok-api23)

Вместо 10 запросов `/video/info?id=X` делать 1 `/video/info?id=A,B,C,...`. Экономия: 1 unit базовый + (N-1) extras. Для 10 видео: 10 запросов vs 10 units (-90% HTTP-вызовов = -90% риска rate-limit).

### 4. Polling-эндпоинты вместо full info

Для счётчиков:
- TikTok: `/api/updated_metadata?id=A,B,C` вместо `/api/user/info`.
- yt-api: `/updated_metadata?id=A,B,C` вместо `/video/info` без `extend`.

Они дешевле и быстрее.

### 5. LLM Prompt Caching

При большом объёме переписываний один и тот же системный промпт = огромная экономия:

- Anthropic: `cache_control: {"type": "ephemeral"}` → 90% скидка на повторы в течение 5 минут.
- При потоке > 100 запросов/час — экономия 80-90% на токенах.

См. [installation-via-api.md](../installation-via-api.md).

### 6. Haiku вместо Sonnet для массового переписывания

Claude Haiku в 5 раз дешевле Sonnet, но на простом переписывании по шаблону разница в качестве небольшая. Ставь Haiku по умолчанию, Sonnet — для премиум-сегмента.

### 7. Не качай большие видео

YouTube long-form — 50-300 MB на видео. Если транскрипция нужна — извлекай только аудио (1-3 MB). Bandwidth-overage > 50 GB = $50+ в месяц.

### 8. Опрос только в активные часы

Аудитория спит — нет смысла опрашивать. Снижение активного окна с 24 до 16 часов = -33% запросов = -33% стоимости.

### 9. Деактивация неактивных авторов

Автор не постит > 30 дней? Снижай частоту на × 5 (раз в 5 часов вместо часа). Если > 90 дней — pause.

## Когда переходить на следующий тариф

Точки перехода (с учётом overage):

### Instagram Looter2

- **Pro $9.90 → Ultra $27.90**: при > 15k req/мес (Pro hard limit, не overage). **Сразу на Ultra если ожидаешь регулярно > 15k.**
- **Ultra → Mega $75.90**: при > 75k req/мес. Точка перехода: $75.90 / $0.001 = 48 000 extras на Ultra → выгоднее Mega при > 123k req/мес.

### TikTok api23

- **Pro $9.99 → Ultra $49.99**: точка перехода ~400k req/мес (на Pro overage $0.0002 × 200k extras = $40 = почти Ultra).
- **Ultra → Mega $99.99**: точка перехода ~1.3M req/мес (overage $0.00015 × 333k = $50).

### YT-API

- **Basic $0 (300 req)** годится только для тестов.
- **Pro $51**: до 1.77M req/мес — большинство юзкейсов покрывает.
- **Ultra $144**: > 1.77M req/мес.

### Threads

- **Basic 50 req/мес** — даже не пытайся, всего на тесты.
- **Pro $19.99 → Ultra $99.99**: когда ожидаешь > 20k req/мес.

## Калькулятор: код для расчёта

```python
def estimate_cost(
    platform: str,
    n_authors: int,
    days: int = 30,
    posts_per_day: float = 1.0,
    p_viral: float = 0.05,
    p_million: float = 0.01,
):
    # Лимиты и цены планов (verified из карточек коннекторов)
    plans = {
        "instagram-looter2": [
            ("Basic", 150, 0, 0),         # hard limit
            ("Pro", 15_000, 9.90, 0),      # hard limit
            ("Ultra", 75_000, 27.90, 0.001),
            ("Mega", 250_000, 75.90, 0.0005),
        ],
        "tiktok-api23": [
            ("Basic", 100, 0, 0),
            ("Pro", 200_000, 9.99, 0.0002),
            ("Ultra", 1_000_000, 49.99, 0.00015),
            ("Mega", 2_000_000, 99.99, 0.0001),
        ],
        "yt-api": [
            ("Basic", 300, 0, 0),
            ("Pro", 1_770_500, 51, 0),  # hard
            ("Ultra", 5_000_000, 144, 0),
            ("Mega", 8_400_000, 240, 0.00003),
        ],
        "threads-api4": [
            ("Basic", 50, 0, 0),
            ("Pro", 20_000, 19.99, 0.0035),
            ("Ultra", 100_000, 99.99, 0.002),
            ("Mega", 500_000, 299.99, 0.001),
        ],
    }
    requests_per_author_per_day = 25  # 1 профиль + 24 видео ленты
    monthly_requests = n_authors * requests_per_author_per_day * days

    # Подбираем план
    chosen = None
    for name, quota, price, overage in plans[platform]:
        if monthly_requests <= quota or overage > 0:
            chosen = (name, quota, price, overage)
            break

    if chosen is None:
        chosen = plans[platform][-1]  # Mega

    name, quota, price, overage = chosen
    extras = max(0, monthly_requests - quota)
    api_cost = price + extras * overage

    # Виральные
    posts_per_month = n_authors * days * posts_per_day
    virals = posts_per_month * p_viral
    millions = virals * p_million

    cost_per_million = api_cost / millions if millions else float("inf")

    return {
        "platform": platform,
        "n_authors": n_authors,
        "monthly_requests": monthly_requests,
        "plan": name,
        "api_cost_usd": round(api_cost, 2),
        "expected_virals": round(virals, 1),
        "expected_millions": round(millions, 2),
        "cost_per_million_usd": round(cost_per_million, 2),
    }


# Пример
print(estimate_cost("tiktok-api23", n_authors=200))
# {
#   "platform": "tiktok-api23",
#   "n_authors": 200,
#   "monthly_requests": 150000,
#   "plan": "Pro",
#   "api_cost_usd": 9.99,
#   "expected_virals": 300.0,
#   "expected_millions": 3.0,
#   "cost_per_million_usd": 3.33,
# }
```

## Чеклист "когда не нужен апгрейд тарифа"

- [ ] Я внедрил кэш на 50-55 минут?
- [ ] Я использую `fields` в Instagram?
- [ ] Я использую multi-id batch в YT-API/TikTok?
- [ ] Я отключил мониторинг неактивных авторов?
- [ ] Я опрашиваю только в активные часы?
- [ ] Я не делаю `extend=1` / `local=1` где не нужно?
- [ ] Я кэширую профили на сутки?
- [ ] Я проверил что hard-limit Basic/Pro меня не остановит?

Если все галочки — следующий тариф реально нужен.
