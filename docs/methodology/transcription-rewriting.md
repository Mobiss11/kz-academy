# Транскрипция и переписывание скриптов

> 🎯 **Цель:** из найденного виральногo видео получить (а) текстовую расшифровку речи и (б) переписанный скрипт под твою аудиторию/нишу/язык.

## Pipeline

```
viral_video.mp4
   │
   ▼  (Этап 1)
┌─────────────────┐
│  Whisper API    │  ← primary
│  AssemblyAI     │  ← fallback
└─────────────────┘
   │
   ▼  text + timestamps
viral_video.transcription
   │
   ▼  (Этап 2)
┌─────────────────┐
│  GPT-4 / Claude │
│  с промптом     │
└─────────────────┘
   │
   ▼
viral_video.rewritten_script
   │
   ▼  (Этап 3)
Уведомление в Telegram + UI
```

## Этап 1. Транскрипция (audio → text)

### Сервисы (порядок приоритета)

| Сервис | Цена | Качество | Auto-detect language | Лимиты |
|---|---|---|---|---|
| **OpenAI Whisper** (`whisper-1`) | $0.006 / минута | Отличное для 50+ языков | ✅ да | 25 MB/файл |
| **AssemblyAI** | $0.012 / минута | Топ для английского | ✅ да | без лимита размера |
| **Yandex SpeechKit** | от ₽0.10/секунда | Топ для русского | ❌ только указанный язык | RU/EN/Kazakh |
| **Google Speech-to-Text** | $0.006 / 15 сек | Хорошее, дороже | ✅ да | 480 минут/файл |

> 💡 **Стартуй с Whisper** — самый дешёвый и универсальный. AssemblyAI как fallback для случаев когда Whisper падает на больших файлах.

### Whisper API: минимальный пример

```python
from openai import OpenAI

client = OpenAI()  # OPENAI_API_KEY

with open("viral_video.mp4", "rb") as f:
    transcript = client.audio.transcriptions.create(
        model="whisper-1",
        file=f,
        # язык auto-detect, можно явно: language="ru"
        response_format="verbose_json",  # с timestamps по сегментам
    )

print(transcript.text)
print(transcript.segments)  # массив {start, end, text}
```

### Когда нужны timestamps

`response_format="verbose_json"` даёт сегменты с временными метками. Полезно для:
- Хука (первые 3-5 сек) — отдельный анализ.
- Резки видео по фразам.
- Subтитров.

Если нужен plain текст — `response_format="text"` (быстрее, дешевле обработать).

### Подготовка видео перед транскрипцией

Whisper лимит — **25 MB на файл**. Большие видео (> 5 минут / 50 MB) надо:

**Вариант A**: Извлечь только аудио (ffmpeg):
```bash
ffmpeg -i video.mp4 -vn -acodec mp3 -ab 64k audio.mp3
# 64 kbps mp3 → ~480 KB/минуту → влезет 50 минут в 25 MB
```

**Вариант B**: Разбить на куски:
```bash
ffmpeg -i video.mp4 -f segment -segment_time 600 -c copy chunk_%03d.mp4
# Куски по 10 минут → транскрибируешь каждый → склеиваешь
```

> 💡 Для Reels/Shorts/TikTok (≤ 60 сек) — обычно 1-3 MB, ничего готовить не надо.

### Auto-detect языка

По умолчанию Whisper детектит язык сам. Точность ~95% для основных языков.

Если ты знаешь что весь твой контент на русском — указывай `language="ru"` явно. Это:
- Чуть точнее (~98%).
- Чуть быстрее (нет фазы детекта).

### Таймауты и retry

```python
client = OpenAI(timeout=300.0)  # 5 минут — длинные видео

# Retry на ошибку
import tenacity

@tenacity.retry(
    wait=tenacity.wait_exponential(min=2, max=30),
    stop=tenacity.stop_after_attempt(3),
)
def transcribe(file_path):
    with open(file_path, "rb") as f:
        return client.audio.transcriptions.create(model="whisper-1", file=f)
```

### Триггеры запуска транскрипции

3 варианта:

| Триггер | Когда |
|---|---|
| **Auto** | Сразу после попадания видео в `viral_content` (приоритет: urgent/high) |
| **Manual** | Кнопка "Транскрибировать" в UI |
| **Batch** | Раз в сутки, для всех видео что не транскрибированы |

Приоритеты в очереди транскрипции:

| Приоритет | Кому |
|---|---|
| 10 (urgent) | VIP-юзеры |
| 7 (high) | Видео > 100k просмотров |
| 5 (normal) | Обычные виральные |
| 3 (low) | Старые / batch-задачи |

## Этап 2. Переписывание (text → adapted script)

### LLM выбор

| LLM | Когда |
|---|---|
| **GPT-4o** / **GPT-4 Turbo** (через Assistants API) | Универсал, RU отлично |
| **Claude Sonnet 4.6** (Anthropic SDK) | Лучше держит стиль и инструкции, длинные промпты |
| **Claude Haiku** | Бюджетный, быстрый, для массовой обработки |
| **GPT-3.5** | Дешёвый, но качество перевода/адаптации хуже |

> 💡 **Для премиум-сегмента — Claude Sonnet с prompt caching** (см. [installation-via-api.md](../installation-via-api.md)). Кэш делает повторные запросы с тем же системным промптом в 10 раз дешевле.

### Базовый промпт

Готовый промпт, проверенный на тысячах Reels (адаптируй под свою нишу):

```
Ты — лучший сценарист коротких видео (Reels / Shorts / TikTok).

Твоя задача: переписать этот текстовый сценарий под российскую/{локаль} аудиторию.

ПРАВИЛА:

1. Не выдумывай несуществующие факты. Если автор сказал X — оставь X.
   Можешь добавить свои примеры до или после, но ключевую мысль не искажай.

2. Замени культурные привязки на местные:
   - Иностранные фильмы → известные местной аудитории.
   - Иностранные компании/бренды → местные аналоги.
   - Иностранные имена → имена/типажи местные.

3. Длина итогового сценария — не более 40 секунд речи (≈80-100 слов).

4. Структура:
   - Хук в первой фразе (вопрос / шок / интрига).
   - Тело — основная мысль, разбитая на 2-3 коротких блока.
   - Финал — побуждение / вывод / cliffhanger.

5. Стиль:
   - Разговорный, как будто рассказываешь другу.
   - Никакого "канцелярита".
   - Используй местные сленги/мемы (умеренно).

6. ВЫВОДИ ТОЛЬКО ГОТОВЫЙ СЦЕНАРИЙ. Без пояснений, без markdown,
   без "Вот ваш скрипт:". Просто текст для озвучки.

Исходный сценарий:
---
{transcription}
---
```

### Действия (actions)

Помимо общего "переписать", полезны более точечные:

| Action | Промпт-добавка |
|---|---|
| `rewrite` | "Полностью переписать под Reels/Shorts формат" (базовый) |
| `add_depth` | "Не меняй основную идею. Добавь 1-2 конкретных примера или цифру для убедительности." |
| `change_cta` | "Сохрани содержание. Замени call-to-action в финале на: '{new_cta}'." |
| `shorten` | "Сократи до 25 секунд (50 слов), сохранив главную мысль." |
| `expand_hook` | "Переделай первую фразу — сделай хук сильнее. Остальное оставь." |
| `make_listicle` | "Преврати в формат '5 признаков того, что...' / '3 ошибки...'." |
| `change_tone` | "Перепиши в тоне '{новый_тон}': юмор / экспертный / эмоциональный." |

### Изоляция thread/conversation

**Каждый запрос — новый thread.** НЕ переиспользуй контекст между разными видео.

Почему:
- Контекст из одного видео может протечь в другое (LLM начнёт мешать темы).
- Разные стили / ниши конфликтуют.
- При ошибке/повторе — чистая попытка лучше.

```python
# ❌ Плохо
thread_id = "always_same"
client.beta.threads.messages.create(thread_id=thread_id, ...)

# ✅ Хорошо
thread = client.beta.threads.create()
client.beta.threads.messages.create(thread_id=thread.id, ...)
# В конце удалить:
client.beta.threads.delete(thread.id)
```

### Таймаут и retry

```python
import openai

try:
    response = client.chat.completions.create(
        model="gpt-4o",
        timeout=300,  # 5 минут
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": transcription},
        ],
    )
except openai.APITimeoutError:
    # повторить или fallback на меньшую модель
    ...
```

### Структурированный вывод

Если хочешь больше чем просто текст — например, отдельно хук/тело/финал:

```python
response_format={
    "type": "json_schema",
    "json_schema": {
        "name": "script",
        "schema": {
            "type": "object",
            "properties": {
                "hook": {"type": "string", "description": "Первая фраза, до 15 слов"},
                "body": {"type": "array", "items": {"type": "string"},
                         "description": "2-3 блока, каждый до 30 слов"},
                "cta": {"type": "string", "description": "Финал, до 15 слов"},
                "estimated_duration_seconds": {"type": "number"},
            },
            "required": ["hook", "body", "cta"],
        },
    },
}
```

Удобно для UI: показываешь хук отдельно жирным, тело пунктами, CTA в рамке.

## Этап 3. Доставка результата

После генерации скрипта:

1. **Сохранить** в `viral_content.rewritten_script` (БД).
2. **Уведомить** автора заявки в Telegram (или email):

```python
import telegram

bot = telegram.Bot(TELEGRAM_BOT_TOKEN)
await bot.send_message(
    chat_id=user.telegram_id,
    text=f"✅ Готов сценарий для @{author.username}\n\n"
         f"Хук: {script.hook}\n\n"
         f"Полный скрипт в кабинете: {url}",
)
```

3. **Логировать** для аналитики (что переписали, какой action, сколько токенов).

## Стоимость pipeline

Грубая прикидка для одного 60-секундного Reel:

| Этап | Стоимость |
|---|---|
| Транскрипция (Whisper) | $0.006 (60 сек = 1 мин) |
| Переписывание (GPT-4o, ~3000 input + 1000 output токенов) | ~$0.025 |
| Переписывание (Claude Sonnet 4.6) | ~$0.020 |
| **Итого на видео** | **~$0.03** |

При 100 видео в день = **$3/день = $90/мес**.

При 1000 видео в день = **$30/день = $900/мес** — добавь prompt caching и Haiku, выйдешь на $300.

## Тонкости и ловушки

### 1. Whisper галлюцинирует на тишине

Если в видео долгая пауза без речи — Whisper может "придумать" что-то. Защита: фильтруй сегменты с `no_speech_prob > 0.8`.

### 2. Длинные видео обрабатывай частями

Если транскрипция > 8000 токенов — LLM может потерять смысл. Решения:
- Разбей по сегментам и обработай раздельно.
- Сначала суммаризуй, потом переписывай саммари.
- Используй модель с большим контекстом (Claude — 200k, Gemini — 1M).

### 3. LLM иногда добавляет воду

Особенно GPT-3.5. Защита: явно в промпте — "макс 100 слов", "ВЫВОДИ ТОЛЬКО ТЕКСТ". И постпроверка на длину:

```python
words = result.split()
if len(words) > 110:
    # перевыполни с более жёстким лимитом
    ...
```

### 4. Нецензурщина / запрещённый контент

LLM может отказаться переписать матерный или политический ролик. Защита:
- Pre-filter транскрипции (заменяй грубые слова на эвфемизмы перед отправкой в LLM).
- Используй модель с менее строгими фильтрами (Mistral, локальные Llama).

### 5. Авторские права

Переписанный скрипт — производное произведение. Юридически: можно для inspiration / референса, но прямую публикацию рискованно для коммерческого использования. Дисклеймер в UI обязателен.

### 6. Кэш промптов экономит до 90%

Если у тебя одинаковый системный промпт на тысячи запросов — используй prompt caching:

- **Anthropic**: `cache_control: {"type": "ephemeral"}` (5 минут TTL).
- **OpenAI** (Azure / некоторые): автоматическое кэширование.
- **Gemini**: `cached_contents` API.

Подробности — в [installation-via-api.md](../installation-via-api.md).

## Шаблоны промптов под популярные ниши

### Education / Talking-head

```
Перепиши под формат "обучающее видео в Reels". Стиль:
- Сразу проблема: "Если ты {ситуация} — этот ролик для тебя."
- 3 пункта решения, пронумерованных.
- Финал: "Сохрани, чтобы не забыть."
```

### Comedy / Memes

```
Адаптируй под русский юмор, отсылки на местные мемы и понятные ситуации.
Сохрани темп шуток. Если оригинал на английском — найди эквивалент русского
сленга/мемов вместо буквального перевода.
```

### Lifestyle / Travel

```
Перепиши в стиле "путевой заметки от друга":
- Личное наблюдение в начале ("Я был в X, и заметил что...").
- Конкретные детали (название места, цена, время).
- Эмоциональный финал.
```

### Business / Finance

```
Перепиши под российский контекст:
- Замени иностранные компании на российские аналоги (Тинькофф, Сбер, Яндекс).
- Цифры конвертируй в рубли по курсу 95 ₽/$.
- Упомяни местные особенности налогов / банков.
- Тон — экспертный, но без занудства.
```
