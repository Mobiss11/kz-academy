# Алгоритм виральной детекции

> 🎯 **Цель:** научиться отличать **виральное видео** от просто популярного. Виральное — это аномальный всплеск роста, который имеет смысл анализировать и переписывать.

## Проблема "наивного" подхода

Если просто фильтровать `views > 100_000`:

❌ В выдачу попадают все видео мега-блогеров (у них 100k за час — норма).
❌ Пропускаются "выстрелы" у новых блогеров, у которых 50k — это феномен.
❌ Не различаешь "набирает скорость прямо сейчас" vs "набрал 200k за полгода".

Нужно **3 фильтра одновременно**.

## Тройной фильтр виральности

Видео считается виральным когда **ВСЕ ТРИ** условия выполняются:

### Фильтр 1. Абсолютный порог по платформе

| Платформа | Порог по умолчанию | Можно настроить per-user |
|---|---|---|
| Instagram | 100 000 просмотров | `instagram_viral_threshold` |
| YouTube | 50 000 просмотров | `youtube_viral_threshold` |
| TikTok | 100 000 просмотров | `tiktok_viral_threshold` |
| Threads | 30 000 просмотров | `threads_viral_threshold` |

Это страховка от "у него 200 подписчиков, видео набрало 1000 просмотров — это виральность 500%, но реально никто не увидит". Минимальный потолок для приличного охвата.

### Фильтр 2. Превышение baseline автора

**Baseline** — медиана просмотров последних 10-30 видео этого автора (без учёта аномалий).

```
baseline = median(last_30_videos.views)

video.views > baseline × MULTIPLIER
```

Рекомендуемые множители:

| Уровень виральности | Множитель | Когда использовать |
|---|---|---|
| Свеж (свеже-горячий) | × 3 | мониторинг для быстрых реакций |
| Уверенный | × 5 | основной фильтр для каталога |
| Топ | × 10 | для "хайлайтов недели/месяца" |

Пример: автор обычно 50k просмотров (baseline). Виральное по умолчанию (×5) = 250k+.

> 💡 **Используй медиану, не среднее.** Один взлёт ломает среднее, медиана устойчива.

### Фильтр 3. Скорость набора (ССН — средняя скорость набора)

Самый важный фильтр. Анализирует **динамику**, не статичный снимок.

**Окно**: 6 часов (по умолчанию). Можно от 3 до 24 часов в зависимости от платформы.

```
ССН = (views_сейчас - views_6_часов_назад) / 6
```

**Пороги** (для Reels/Shorts/TikTok):

| Уровень | ССН (просмотров/час) | Когда срабатывает |
|---|---|---|
| **ССН** (базовый) | ≥ 100 | средняя виральность |
| **ССН1** (90% базового) | ≥ 90 | для менее активных авторов |
| **ССН2** (80% базового) | ≥ 80 | для начинающих авторов |

**Логика**: автор с 50k базовой средней vs автор с 5k — у первого видео может пробивать ССН пробить легче. Использование ССН1/ССН2 даёт шанс маленьким каналам.

**Для длинных видео** (YouTube long-form): пересчитай в просмотрах/день. Виральное long-form ≈ 10k+ просмотров/день стабильно.

### Композитное правило

```python
def is_viral(video, author, platform):
    # Фильтр 1: абсолют
    if video.views < ABSOLUTE_THRESHOLD[platform]:
        return False

    # Фильтр 2: baseline
    baseline = median(author.last_30_videos.views)
    if video.views < baseline * 5:  # x5 для уверенной виральности
        return False

    # Фильтр 3: ССН
    ssn = (video.views_now - video.views_6h_ago) / 6
    if ssn < SSN_THRESHOLDS["ССН"]:  # 100 просмотров/час
        return False

    return True
```

## Окна и интервалы

| Параметр | По умолчанию | Можно менять |
|---|---|---|
| Окно ССН | 6 часов | 3-24 ч |
| Интервал проверки | 1 час | 30 мин - 6 ч |
| Retention сигналов | 48 часов | 24-72 ч |
| История baseline | 30 дней | 14-90 дней |

> 💡 **Меньше окно = быстрее обнаружение, но больше шума.** Если у тебя real-time дашборд — 3 часа. Если отчёт раз в сутки — 12-24 часа.

## Реальный flow детекции

```
Каждый час:
  1. Для каждого активного автора:
     1.1. Подтянуть последние 30 видео (через эндпоинт мониторинга).
     1.2. Записать новую точку в views_history каждого видео.

  2. Для каждого видео < 48 часов от загрузки:
     2.1. Посчитать ССН = (views_now - views_6h_ago) / 6.
     2.2. Сравнить с порогами ССН/ССН1/ССН2.
     2.3. Если ≥ порог — записать в trend_signals (тип сигнала).

  3. Для каждого видео из trend_signals:
     3.1. Подтянуть baseline автора (медиана последних 30 видео).
     3.2. Проверить: views > baseline × 5 И views > ABSOLUTE_THRESHOLD?
     3.3. Если да → ViralContent (двигаем дальше по pipeline).

  4. Cleanup: удалить trend_signals старше 48 часов.
```

## Хранение временных рядов

Для каждого видео нужна история просмотров. 2 варианта:

### Вариант A: JSON-поле в строке видео

```sql
CREATE TABLE videos (
  id BIGINT PRIMARY KEY,
  author_id BIGINT,
  views_history JSONB,  -- [{ts: 2026-04-28T10:00, v: 12000}, ...]
  ...
);
```

Плюсы: проще, всё в одной строке.
Минусы: при росте JSON растёт, обновление через `views_history = views_history || '[{...}]'` — медленнее на больших объёмах.

### Вариант B: отдельная таблица time-series

```sql
CREATE TABLE video_views_snapshots (
  video_id BIGINT,
  snapshot_at TIMESTAMP,
  views BIGINT,
  PRIMARY KEY (video_id, snapshot_at)
);
CREATE INDEX idx_video_snapshot_at ON video_views_snapshots (snapshot_at);
```

Плюсы: легко чистить старые точки, оптимально для time-series queries.
Минусы: больше JOIN'ов.

**Рекомендация**: для < 10k активных видео — JSON. Для > 10k — отдельная таблица.

## Cleanup старых сигналов

Через 48 часов виральный сигнал теряет актуальность (видео либо уже разогналось до взрыва, либо забыто). Чисти:

```sql
DELETE FROM trend_signals WHERE created_at < NOW() - INTERVAL '48 hours';
```

Делай это раз в сутки, ночью. Иначе таблица распухнет.

## Тонкости и ловушки

### Спайки от внешних источников

Иногда видео бустится извне (репост в большой Telegram-канал, накрутка). ССН резко взлетает, но это не органика.

**Защита**: проверяй **продолжительность** ССН. Если ССН ≥ порог только в 1 окне (6 часов), а потом сразу падает — скорее всего бустинг. Если держится 2-3 окна — органическая виральность.

### Холодный старт автора

Новый автор → нет 30 видео для baseline → не на чем считать.

**Защита**: для авторов с < 5 видео используй платформенные дефолты:
- Instagram: baseline = 5 000.
- YouTube: baseline = 2 000.
- TikTok: baseline = 10 000.

Пересчитывай baseline когда наберётся достаточно видео.

### Платформенные особенности

| Платформа | Особенности |
|---|---|
| **TikTok** | алгоритмы быстрые — окно 3-6 часов оптимально. ССН порог можно повышать. |
| **Instagram Reels** | органика медленнее — окно 12-24 часа. Stories не учитывай. |
| **YouTube Shorts** | окно 12 часов. Long-form — день. |
| **YouTube long-form** | смотри в просмотрах/день, не в час. |

### Удалённые/приватные видео

Если автор удалил видео — внезапно `views = null` или 404. Не считай это нулём.

**Защита**: при ошибке API не обновляй `views_history`. Помечай видео как `last_error_at`. Если 3 раза подряд — flag `is_deleted = true`.

## Numerical Cheat Sheet

Готовые цифры для копирования:

```yaml
# Абсолютные пороги (минимум просмотров чтобы вообще рассматривать)
absolute_threshold:
  instagram: 100_000
  youtube_shorts: 50_000
  youtube_longform: 25_000
  tiktok: 100_000
  threads: 30_000

# Множитель к baseline (медиана за 30 дней)
baseline_multiplier:
  fresh:     3   # для оперативного мониторинга
  confident: 5   # стандартный
  top:       10  # для "хайлайтов"

# Скорость набора (просмотров/час за 6-часовое окно)
ssn_thresholds:
  ССН:  100  # базовый
  ССН1:  90  # для менее активных
  ССН2:  80  # для холодного старта

# Окна (часов)
windows:
  ssn_window: 6
  detection_interval: 1
  retention: 48

# Холодный старт baseline
cold_start_baseline:
  instagram: 5_000
  youtube: 2_000
  tiktok: 10_000

# История baseline
baseline_period_days: 30
```

## Тестирование алгоритма

Чтобы проверить что фильтр работает:

1. Возьми **10 заведомо виральных** видео (выстрелы, которые ты знаешь).
2. Возьми **10 проходных** видео тех же авторов.
3. Прогон через свой фильтр. Должны попасть **только** виральные.

Если виральные не ловятся — снижай пороги. Если проходные ложно срабатывают — повышай.

**Tuning по нише**:
- Edu/Talking-head: пороги выше (медленнее набирают).
- Comedy/Memes: пороги ниже (быстрый pickup).
- Trends/Челленджи: окно меньше (3 часа).
