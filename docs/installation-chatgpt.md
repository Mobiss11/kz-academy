# Подключение к ChatGPT

В ChatGPT нет прямого аналога скиллов, но есть два способа дать модели контекст репозитория.

## Вариант 1: ChatGPT Projects (рекомендуется)

1. Создай новый Project в ChatGPT (нужен Plus/Team/Enterprise).
2. В **Files** загрузи `SKILL.md` и нужные карточки из `connectors/`.
3. В **Instructions** добавь:

   ```
   Этот проект помогает работать с RapidAPI. В файлах загружены SKILL.md
   с общими правилами и карточки коннекторов. Когда я спрашиваю про
   любой API из RapidAPI — используй эти файлы как источник истины:
   правильные хедеры, эндпоинты, параметры. Никогда не выдумывай поля.
   ```

## Вариант 2: Custom GPT

Если хочешь сделать общий инструмент для учеников:

1. Создай Custom GPT в [GPT Builder](https://chatgpt.com/gpts/editor).
2. В **Knowledge** загрузи `SKILL.md` + все карточки `connectors/*.md`.
3. В **Instructions**:

   ```
   Ты — помощник по работе с RapidAPI. Используй файлы из knowledge:
   SKILL.md (общие правила) и connectors/*.md (специфика конкретных API).

   Когда пользователь упоминает любой API из RapidAPI:
   1. Открой соответствующую карточку в connectors/.
   2. Сформируй код на Python или другом языке по спецификации.
   3. Никогда не выдумывай имена параметров — они должны быть из карточки.
   4. Всегда добавляй хедеры X-RapidAPI-Key и X-RapidAPI-Host.

   Если карточки нужного коннектора нет — предупреди и используй общие
   правила из SKILL.md.
   ```

4. Опубликуй внутри workspace и поделись ссылкой с учениками.

## Ограничения

- Knowledge у Custom GPT обновляется только пересохранением — нельзя автоматически тянуть из git. Решение: при апдейте репо вручную перезагружай файлы.
- Без code interpreter ChatGPT не сможет реально вызвать API — только сгенерировать код.
