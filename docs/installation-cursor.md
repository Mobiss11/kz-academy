# Подключение к Cursor

В [Cursor](https://cursor.sh/) есть `.cursorrules` (старый формат) и `.cursor/rules/` (новый, MDC-формат). Скилл можно использовать обоими способами.

## Вариант 1: через `.cursor/rules/` (рекомендуется)

1. Склонируй репозиторий куда удобно (например, рядом с проектом):

   ```bash
   git clone https://github.com/Mobiss11/kz-academy.git
   ```

2. В корне твоего проекта создай `.cursor/rules/rapidapi.mdc`:

   ```markdown
   ---
   description: Используй для любых задач с RapidAPI — формирование запросов, отладка ошибок 401/403/429, работа с конкретными коннекторами (yt-api и др.)
   globs: ["**/*.py", "**/*.ts", "**/*.js"]
   alwaysApply: false
   ---

   # RapidAPI Helper

   Когда задача связана с RapidAPI:

   1. Сначала прочитай @../kz-academy/SKILL.md — там общие правила.
   2. Затем найди карточку нужного коннектора в @../kz-academy/connectors/.
   3. При создании кода используй обвязку из @../kz-academy/examples/common.py.

   Ключевые правила:
   - Всегда передавай оба хедера: X-RapidAPI-Key и X-RapidAPI-Host.
   - Никогда не хардкодь ключ — только через переменные окружения.
   - При 403 — проверь хедер Host и подписку на API.
   ```

   Поправь пути `@../kz-academy/...` под фактическое расположение клона относительно твоего проекта.

## Вариант 2: через `.cursorrules` (legacy)

В корне проекта создай `.cursorrules` и вставь туда содержимое `SKILL.md` плюс ссылку на нужные карточки коннекторов. Минус — нужно вручную поддерживать содержимое в актуальном состоянии.

## Проверка

В Cursor открой Python-файл и в чате спроси:

> Напиши функцию, которая ищет видео через yt-api на RapidAPI

Cursor должен использовать правила и сослаться на содержимое карточки.
